NAME 

vlibDate - php class enabling you to handle dates from year 0001-9999 and in multiple languages. package: vLIB

SYNOPSIS 

   include "./vLIB/vlibDate.php";
   $date = new vlibDate();
   $today = $date->now();

   // now we loop through all languages and display today's date
   foreach ($date->accepted_langs as $lang) {
       $date->setLang ($lang);
       echo $date->formatDate($today, '%A %d %B %Y<br>');
   }

If everything goes well, you should see today's date being displayed in all of the languages supported by this
version of vlibDate.


DESCRIPTION 

vlibDate is a class for displaying and manipulating dates from year 0001-9999.
vLIBDate not only makes this a posibility but makes it an easy task with many different functions to enable
you to do all kinds of date manipulation.
vlibDate also offers you a simple way of displaying dates in multiple languages (SUPPORTED LANGUAGES).


MOTIVATION 

Whilst recently working on a project I needed a way of verifying some Dates of Birth.
Of course, it isn't possible to do anything with dates before 1970 on 32-bit systems, and I had used the
perl CPAN package Date::Calc which contains many methods for manipulating dates.
I also liked some of the functions used in MySQL so I incorporated them as well.

And so this is why vlibDate was created, it's a sort of hybrid between a lite version of Perl's Date::Calc and
MySQL's Date functions.

  • French (fr)

  • Portuguese (pt)
  • Italian (it)

  • Danish (da)
  • Norwegian (no)
  • Swedish (sv)

If you would like to contribute a language then please send an email to vlib@activefish.com with the days of the
week, months of the year, and if possible the suffixes to the days of the month. If the language you would like
to add has different weekday abbreviations than the first few letter (as in portuguese) then please send these
as well.




IMPORTANT 

Just a few of points before you get into vlibDate.

1) Throughout the documentation you will come across quite often the String type timestamp.
    This is not a unix timestamp but a 'vlibDate timestamp'. vlibDate timestamps are simply a
    string in 'YYYY-MM-DD' format, i.e. 2002-01-25. If a unix timestamp is required the unixtime
    will be used as the type definition.

2) In many functions, you can define a format for the string to be returned as. This format uses
    similar syntax to the native php function strftime(), however not all formats are supported.

    Here is a list of the formatting options supported.

    * %a abbreviated weekday name (Sun, Mon, Tue)
    * %A full weekday name (Sunday, Monday, Tuesday)
    * %b abbreviated month name (Jan, Feb, Mar)
    * %B full month name (January, February, March)
    * %d day of month (range 00 to 31)
    * %e day of month, single digit (range 0 to 31)
    * %E number of days since unspecified epoch (integer)
       (%E is useful for passing a date in a URL as an integer value.
        Then use decodeDate() to convert back to a date.)
    * %j day of year (range 001 to 366)
    * %m month as decimal number (range 1 to 12)
    * %n newline character (\n)
    * %s ordinal suffix for day of month
    * %t tab character (\t)
    * %w weekday as decimal (0 = Sunday)
    * %U week number of current year, first sunday as first week
    * %y year as decimal (range 00 to 99)
    * %Y year as decimal including century (range 0000 to 9999)
    * %% literal '%'

example of use:
  $date = new vlibDate('en');
  echo $date->formatDate('2002-02-15', '%A %e%s of %B, %Y');

3) To set a default language for this class, you must edit the ini.php file.


vlibDate 

vlibDate() 

Called upon instantiation of the class, this loads the default language or the language specified by lang:

  vlibDate ([string lang])

The lang parameter must be the code of one of the supported languages.
See the SUPPORTED LANGUAGES section:

   $date = new vlibDate('es'); // load spanish as the default language

NB: You can specifiy a language by default in your ini.php file which will be used if lang is not specified.


formatDate()  

  formatDate (string timestamp, string format)


Formats the date like strftime(). Returns a string in the specifies format for the given timestamp.

The timestamp parameter is a vlibDate timestamp. For more information on this and the format parameter, please
see the IMPORTANT section.

$date->formatDate('2002-02-15', '%A %e%s of %B, %Y');


getMonthFullname()  

  getMonthFullname (string timestamp)

Returns the full name of the month for the given timestamp.

The timestamp parameter is a vlibDate timestamp. For more information on this, please see the IMPORTANT section.

$date->getMonthFullname('2002-04-15'); // returns 'April'

 

getMonthAbbrname()  

  getMonthAbbrname (string timestamp  [, int length=3])

Returns the abbreviated name of the month for the given timestamp.

The timestamp parameter is a vlibDate timestamp. For more information on this, please see the IMPORTANT section.
The length parameter is the length of the sub string (defaults to 3).

$date->getMonthAbbrname('2002-05-15'); // returns 'Apr'

 

getWeekdayFullname()  

  getWeekdayFullname (string timestamp)

Returns the full name of the weekday for the given timestamp.

The timestamp parameter is a vlibDate timestamp. For more information on this, please see the IMPORTANT section.

$date->getWeekdayFullname('2002-04-15'); // returns 'Monday'

 

getWeekdayAbbrname()  

  getWeekdayAbbrname (string timestamp [, int length=3])

Returns the abbreviated name of the weekday for the given timestamp.

The timestamp parameter is a vlibDate timestamp. For more information on this, please see the IMPORTANT section.
The length parameter is the length of the sub string (defaults to 3).

$date->getWeekdayAbbrname('2002-05-15', 4); // returns 'Mond'

 

now()  

  now ()

Returns a vlibDate timestamp for the current date.

The returned timestamp is a vlibDate timestamp. For more information on this, please see the IMPORTANT section. 

   $today = $date->now(); // $today equals '2002-04-28' for example.


getSuffix()  

  getSuffix(string timestamp)

Returns the suffix for the day of the month, i.e. 03 = rd (in english).

The timestamp parameter is a vlibDate timestamp. For more information on this, please see the IMPORTANT section. 

   $suffix = $tmpl->getSuffix('2002-01-01'); // sets $suffix to 'st'


mkTimestamp()  

  mkTimestamp([string year [, string month [, string day]]])

This returns a vlibDate timestamp for the given date values. If any value is omitted, the current value will be used.

The returned timestamp is a vlibDate timestamp. For more information on this, please see the IMPORTANT section.

  $today = mkTimestamp(); // all values omitted, takes current values for everything
  $y2k = mkTimestamp('2000', '01', '01'); // returns '2001-01-01'


fromUnixTime()  

  fromUnixTime(int unixtime)

Returns a vlibDate timestamp for the given unix timestamp.

The returned timestamp is a vlibDate timestamp. For more information on this, please see the IMPORTANT section. 

   $today = $date->fromUnixTime(time());


setLang()  

  setLang(string lang)

The lang parameter is the language code for the desired language.

For a list of supported languages and their codes, please see the SUPPORTED LANGUAGES section.

 

addInterval()  

  addInterval (string timestamp , string interval)

Returns a timestamp with the interval added.
Accepted interval values are DAY[S], WEEK[S], MONTH[S], YEAR[S] and CENTURY [CENTURIES].

The returned timestamp is a vlibDate timestamp. For more information on this, please see the IMPORTANT section. 

  $futuredate = $date->addInterval($date->now(), '2 CENTURIES 5 YEARS 2 MONTHS 3 DAYS');

 

subInterval()  

  subInterval (string timestamp , string interval)

Returns a timestamp with the interval subtracted.
Accepted interval values are DAY[S], WEEK[S], MONTH[S], YEAR[S] and CENTURY [CENTURIES].

The returned timestamp is a vlibDate timestamp. For more information on this, please see the IMPORTANT section. 

  $pastdate = $date->subInterval($date->now(), '2 CENTURIES 5 YEARS 2 MONTHS 3 DAYS');

 

nextDay()  

  nextDay (string timestamp [, string format])

Returns a vlibDate timestamp for the day after timestamp. If format is specified, then that format is returned
instead of the timestamp.

The returned timestamp is a vlibDate timestamp. For more information on this and the format parameter, please
see the IMPORTANT section.

  $tomorrow = $date->nextDay( $date->now() );

 

prevDay()  

  prevDay (string timestamp [, string format])

Returns a vlibDate timestamp for the day before timestamp. If format is specified, then that format is returned
instead of the timestamp.

The returned timestamp is a vlibDate timestamp. For more information on this and the format parameter, please
see the IMPORTANT section.

  $yesterday = $date->prevDay( $date->now() );

 

isFutureDate()  

  isFutureDate (string timestamp)

Returns true if the given timestamp is in the future, false otherwise.

The timestamp parameter is a vlibDate timestamp. For more information on this and the format parameter, please
see the IMPORTANT section.

  $infuture = $date->isFutureDate('2005-01-01');

 

isPastDate()  

  isPastDate (string timestamp)

Returns true if the given timestamp is in the past, false otherwise.

The timestamp parameter is a vlibDate timestamp. For more information, please see the IMPORTANT section.

  $inpast = $date->isPastDate('1999-01-01');

 

isDate()  

  isDate (string timestamp)

Returns true if the given timestamp is valid, false otherwise.

The timestamp parameter is a vlibDate timestamp. For more information, please see the IMPORTANT section. 

  if (!$date->isDate('1999-13-35')) die ('invalid date!');

 

isLeapYear()  

  isLeapYear (string year)

This function returns true if the given year is a leap year. The parameter year can either be a year in format
YYYY or a vlibDate timestamp.

For more information on a vlibDate timestamp, please see the IMPORTANT section. 

  if ($date->isLeapYear('2000')) echo '2000 is a leap year';

 

dayOfWeek()  

  dayOfWeek (string timestamp)

Returns day of week for the given timestamp: 0=Sunday ... 6=Saturday

For more information on a vlibDate timestamp, please see the IMPORTANT section.

 echo $date->dayOfWeek('2002-04-29'); // echo's '1' , Monday

 

weekOfYear()  

  weekOfYear (string timestamp)

Returns week of the year for the given timestamp, the first Sunday is first day of first week of the year.

The timestamp parameter is a vlibDate timestamp. For more information, please see the IMPORTANT section.

  echo $date->weekOfYear('2002-05-01'); //echo's '17'

 

julianDate()  

  julianDate (string timestamp)

Returns number of days since (and including) 1st January of the year in the given timestamp.

The timestamp parameter is a vlibDate timestamp. For more information, please see the IMPORTANT section. 

  echo 'today is day number '.$date->julianDate( $date->now() ).' of the year';


quarterOfYear()  

  quarterOfYear (string timestamp)

Returns quarter of the year for given timestamp.

The timestamp parameter is a vlibDate timestamp. For more information, please see the IMPORTANT section. 

 echo "we're currently in quarter number ".$date->quarterOfYear( $date->now() );

 

getLongYear()  

  getLongYear (string year)

Returns the year in YYYY format for the given year in YY format. 0-49 is considered 21st century, 50-99 is
20th century.

 echo $date->getLongYear('98'); // will echo 1998

 

diffInDays()  

  diffInDays (string timestamp1, string timestamp2)

Returns number of days between two given timestamps.

The timestamp parameters are vlibDate timestamps. For more information, please see the IMPORTANT section. 

  $today = $date->now();
  $diff = $date->diffInDays($today, '2050-01-01');

 

daysInMonth()  

  daysInMonth (string month [, string year])

Returns the number of days in the month for the parameters month and year.
Alternatively you can pass a timestamp as the month parameter and omit the year parameter for the same result.

The timestamp parameter is a vlibDate timestamp. For more information, please see the IMPORTANT section. 

  echo $date->daysInMonth( $date->now() ); // will echo days in current month

 

dateToDays()  

  dateToDays (string timestamp)

Converts a vlibDate timestamp to a number of days since a long distant epoch. You can use this to store dates
or to pass dates from 1 URL to another, and then to reconvert them use daysToDate().

The timestamp parameter is a vlibDate timestamp. For more information, please see the IMPORTANT section. 

  $encoded = $date->dateToDays( $date->now() );


encodeDate()  

  an alias for dateToDays().

 

daysToDate()  

  daysToDate (string timestamp [, string format])

Converts a number of days since a long distant epoch to a vlibDate timestamp or to a format specified by the
format parameter.

The timestamp parameter is a vlibDate timestamp. For more information, please see the IMPORTANT section. 

  echo $date->dayToDate($encoded);


decodeDate()  

  an alias for daysToDate().

  getYear()

getMonth()  

  getMonth()

  getDay()
